# Windows 命令行 Git 提交信息乱码解决方案

## 问题描述

在 Windows PowerShell 中使用 `git commit -m "中文信息"` 时，提交信息会出现乱码，导致 GitHub 上显示异常。

## 解决方案

### 方法一：使用文件方式提交（推荐）

使用 `-F` 参数从文件读取提交信息，避免命令行编码问题：

1. **创建提交信息文件**（UTF-8 编码）：
   ```bash
   # 创建 commit_msg.txt 文件，内容为：
   v0.3.0: 添加全屏时钟功能
   
   - 新增全屏模式下可拖动时钟显示
   - 支持边缘吸附功能
   - 鼠标靠近隐藏的时钟时自动滑出显示
   ```

2. **使用文件提交**：
   ```bash
   git commit -F commit_msg.txt
   ```

3. **提交后删除临时文件**：
   ```bash
   del commit_msg.txt
   ```
